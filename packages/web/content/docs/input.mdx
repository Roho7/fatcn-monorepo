# Input

A button component that follows best practices for accessibility and provides multiple variants for different use cases.

## Props

| Prop             | Type                                                         | Default     | Description                                                |
| ---------------- | ------------------------------------------------------------ | ----------- | ---------------------------------------------------------- |
| `label`          | `string`                                                     | -           | The text of the input label                                |
| `className`      | `string`                                                     | -           | Additional CSS classes for the input container             |
| `inputClassName` | `string`                                                     | -           | Additional CSS classes for the input component             |

## Design Guidelines

- Use clear, descriptive labels and placeholders for context.
- Ensure sufficient padding and legible font size for readability.
- Provide visual feedback for focus, hover, and error states.
- Use appropriate input types (e.g., text, password, email) for better functionality.
- Ensure accessible error messaging and validation hints.

## Accessibility

- Multiple Types: Supports text, email, password, number, and more.
- Validation Support: Includes client-side and server-side validation.
- Customizable Styles: Flexible size, colors, and borders.
- Keyboard and Screen Reader Accessible: Includes ARIA attributes for usability.
- Responsive: Adapts seamlessly across screen sizes and devices.
