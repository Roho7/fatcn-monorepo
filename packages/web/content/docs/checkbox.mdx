# Checkbox

A button component that follows best practices for accessibility and provides multiple variants for different use cases.

## Installation 

```bash
npx fatcn-ui add checkbox
```

## Usage

<ComponentPreview name='checkbox' />


## Props

| Prop        | Type                                                         | Default     | Description                                                |
| ----------- | ------------------------------------------------------------ | ----------- | ---------------------------------------------------------- |
| `label`     | `string`                                                     | -           | The text of the checkbox label                            |
| `className` | `string`                                                     | -           | Additional CSS classes                                     |

## Design Guidelines

- Use clear, descriptive labels for each checkbox.
- Maintain sufficient spacing between checkboxes for readability.
- Provide visual feedback for states: checked, unchecked, and indeterminate.
- Ensure focus and hover states are easily noticeable.
- Group related checkboxes with a clear heading or context.

## Accessibility

- Supports checked, unchecked, and indeterminate.
- Fully operable via keyboard navigation.
- Adaptable size, color, and labels.
- Works seamlessly across devices and screen sizes.
- Includes ARIA roles for accessibility.
