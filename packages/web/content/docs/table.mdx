# Table

A Table is a structured UI component used to display data in rows and columns for easy comparison and analysis.


## Usage


## Props

| Prop            | Type                                                         | Default     | Description                                                |
| --------------- | ------------------------------------------------------------ | ----------- | ---------------------------------------------------------- |
| `data`          | `record`                                                     | -           | Shows the number of pages shown at the same time           |
| `itemsPerPage`  | `number`                                                     | -           | Shows the number of pages shown at the same time           |
| `tableClassName`| `string`                                                     | -           | Additional CSS classes                                     |

## Design Guidelines

- Use clear headers to define the context of each column.
- Ensure proper spacing and alignment for readability and consistency.
- Apply zebra striping or borders for better row differentiation.
- Optimize tables for both scrolling and responsive layouts on smaller screens.
- Provide sorting, filtering, or pagination for tables with extensive data.

## Accessibility

- Supports flexible column widths, alignments, and styles.
- Allows sorting, filtering, and inline editing.
- Adapts to smaller screens with horizontal scrolling or stacked views.
- Includes ARIA roles, semantic tags (`<table>`, `<thead>`, `<tbody>`), and keyboard navigation.
- Supports dynamic data updates and integration with APIs.
