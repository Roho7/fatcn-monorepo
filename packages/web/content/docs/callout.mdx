## Props

| Prop        | Type                                                          | Default     | Description                                                |
| ----------- | ------------------------------------------------------------- | ----------- | ---------------------------------------------------------- |
| `variant`   | `'default' \| 'secondary' \| 'warning' \| 'error' \| 'success`| `'default'` | The visual style of the button                             |
| `disabled`  | `boolean`                                                     | `false`     | Whether the button is disabled                             |
| `loading`   | `boolean`                                                     | `false`     | Shows a loading spinner and disables the button            |
| `asChild`   | `boolean`                                                     | `false`     | Merge props onto child element instead of rendering button |
| `className` | `string`                                                      | -           | Additional CSS classes                                     |

## Design Guidelines

- Use concise, actionable text to communicate the message effectively.
- Ensure the title or main message is more prominent than the supporting text.
- Apply sufficient contrast between the callout and its background for visibility.
- Use meaningful icons that align with the context of the message.
- Align text and visuals consistently for a clean appearance.
- Provide adequate padding and margin to make the callout easy to read.
- Use a uniform style for callouts across the interface to maintain coherence.

## Accessibility

The Button component follows WAI-ARIA guidelines:

- Uses native `<button>` element by default
- Properly handles disabled states
- Includes appropriate ARIA attributes
- Supports keyboard navigation
- Maintains sufficient color contrast
