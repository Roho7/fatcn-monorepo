# Dropdown

A Dropdown is a UI element that displays a list of options or actions when triggered, allowing users to select one or perform an action.

## Installation 

```bash
npx fatcn-ui add dropdown
```

## Usage

<ComponentPreview name='dropdown' />

### Options

The dropdown component takes in an array of objects which lets you configure the options and supoptions.
This design choice was taken to allow for more customization in the options, giving you the ability to set 
icons, actions and rightSlots for each option.

```tsx showLineNumbers
{[
    {
    element: 'Profile Settings',
    icon: <UserCircle02Icon className="h-4 w-4" />,
    action: handleSettingsClick,
    suboptions: [
        {
        element: 'Notifications',
        icon: <SchoolBell01Icon className="h-4 w-4" />,
        action: (e) => console.log('Notifications clicked'),
        },
        {
        element: 'Email Preferences',
        icon: <Mail01Icon className="h-4 w-4" />,
        action: (e) => console.log('Email preferences clicked'),
        },
    ]
    },
    {
    element: 'Logout',
    icon: <Logout01Icon className="h-4 w-4" />,
    action: handleLogout,
    rightSlot: <RightAngleIcon className="h-4 w-4" />
    },
]}
```

## Props

| Prop        | Type                                                         | Default     | Description                                                |
| ----------- | ------------------------------------------------------------ | ----------- | ---------------------------------------------------------- |
| `variant`   | `'default' \| 'secondary'`                                   | `'default'` | The visual style of the button                             |
| `alignment` | `'end' \| 'start'  \| 'center'`                              | `'start'`   | The alignment of the dropdown with respect to the trigger  |
| `triggerSize`| `'sm' \| 'md' \| 'lg'`                                      | `'md'`      | The size of the trigger button                             |
| `children`  | `React.ReactNode`                                            | -           | The content of the trigger                                 |
| `options`   | `Record<string, any>`                                        | -           | The options of the dropdown (See usage above)              |
| `rightSlot` | `React.ReactNode`                                            | -           | The icon to show up on the right side of the button (rotates automatically on dropdown open) | 

## Design Guidelines

- Use a clear trigger (e.g., button or icon) with descriptive labels or tooltips.
- Position dropdowns dynamically to avoid clipping or obscuring content.
- Ensure adequate spacing and hover/focus states for each option.
- Limit the number of options to maintain scannability; group options when necessary.
- Allow keyboard navigation and close the dropdown on outside clicks or `Esc`.

## Accessibility

- Can be opened via click, hover, or keyboard actions.
- Adjusts placement based on screen space.
- Supports icons, checkboxes, dividers, and submenus.
- Navigable via arrow keys and dismissible with `Esc`.
- Adapts to different devices and orientations seamlessly.
